Дневная статистика по кампаниям

GET /api/client/statistics/daily

Если в запросе не указаны значения полей dateFrom и dateTo, то в ответе будет статистика за последние 7 дней.

Формат отчёта — CSV.

Чтобы запросить отчёт в формате JSON, добавьте в адрес метода /json и отправьте запрос по адресу /api/client/statistics/daily/json.

Отчёт содержит:

  • идентификатор кампании,
  • название кампании,
  • дата,
  • показы,
  • клики,
  • расход в рублях,
  • заказы в штуках,
  • заказы в рублях.
QUERY PARAMETERS
campaignIds
Array of strings <uint64>

Список идентификаторов кампаний, для которых необходимо подготовить отчёт.

Если поле заполнено, в отчёте будет статистика только по указанным кампаниям. Если поле пустое, в ответе будет статистика по всем кампаниям в указанном промежутке времени.

Необязательное поле.

dateFrom
string

Начальная дата периода отчёта в формате ГГГГ-ММ-ДД. Например: 2019-02-10.

Необязательное поле.

dateTo
string

Конечная дата периода отчёта в формате ГГГГ-ММ-ДД. Например: 2019-02-10.

Необязательное поле.

Получить отчёт по заказам в оплате за заказ — выбранные товары

POST /api/client/statistic/orders/generate

В параметрах запроса from и to укажите промежуток времени.

Performance API асинхронный. Поэтому в результате запроса будет не сам отчёт, а уникальный идентификатор отправленного запроса, с помощью которого можно проверить статус формирования отчёта и скачать отчёт в формате CSV.

Чтобы запросить отчёт в формате JSON, добавьте /json в адрес метода: /api/client/statistic/orders/generate/json.

Отчёт содержит:

  • период отчёта;
  • дату;
  • ID заказа;
  • номер заказа;
  • SKU;
  • SKU продвигаемого товара;
  • артикул;
  • наименование;
  • источник заказа.

Примеры запроса

Content type
application/json
{
  "from": "2019-08-24T14:15:22Z",
  "to": "2019-08-24T14:15:22Z"
}
